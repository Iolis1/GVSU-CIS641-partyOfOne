Project Description

This project aims to simplify and present medical test results using a Large Language Model (LLM) within Electronic Medical Records (EMR) systems. The main goal is to make complex medical data more accessible and understandable for patients through their health portals, improving patient engagement and reducing anxiety by providing clear, patient-friendly summaries of test results.

Prerequisites

  • A system capable of running PHP 8.0 or higher.
  • Apache or Nginx web server configured to serve PHP applications.
  • MySQL 8.0 or higher for database storage.
  • Composer for dependency management (for OpenEMR).
  • Python 3.8 or higher with the following libraries:
    • Flask
    • PyMySQL
    • transformers
    • openai
    • zlib
  • Node.js and npm for frontend dependency management if UI customizations are needed.
  • SSL certificates to ensure HTTPS compliance for secure communications.

Run Instructions

  1. Install all prerequisites mentioned above.
  2. Set up the MySQL database:
    • Create a database and run the provided `schema.sql` script to set up tables.
    • Update the database credentials in the configuration file (`config.json` for middleware).
  3. Set up the PHP server:
    • Place the project files in the Apache/Nginx web server's document root.
    • Ensure the `globals.php` file points to the correct paths for OpenEMR dependencies.
  4. Run the Python middleware:
    • Navigate to the middleware folder.
    • Install the required Python libraries using `pip install -r requirements.txt`.
    • Update the database credentials and api keys in the configuration file you create(`config.json`).
    • Run the Flask application using `python summarization_api.py`.
    • Ensure the middleware is accessible via `http://localhost:5000`.
  5. Launch the application:
    • Open your browser and navigate to the OpenEMR dashboard.
    • Click on "View All Lab Summaries" to generate and review summaries.

Training Information

  • **Training**:
    • To generate a summary follow a few easy steps
      1. Click on the "View All Lab Summaries" button
      2. Click on the "Generate Summary" button
      3. Wait for the summary to be generated
      4. Review the summary and make any necessary edits
      5. Click on the "Approve Summary" button
  • **FAQs**:
    • How do I generate a lab summary?
    • How do I edit and approve a summary?
    • What do I do if the summarization fails?
  • **Troubleshooting**:
    • Check error logs in `logs/errors.log` for middleware issues.
    • Ensure database credentials are correctly configured in the `config.json` file.

Features

  • Automatic summarization of lab results using BioGPT and OpenAI's GPT-4o-mini.
  • User-friendly interface for editing and approving summaries.
  • Secure data handling in compliance with HIPAA and GDPR.
  • Performance-optimized middleware to handle concurrent requests.
  • Backup and recovery support for database data.
