Overview

This document serves as a reference for the software requirements specifications for the Party of One project.

Functional Requirements

  • Professional Access Requirement
    • FR1: The program shall allow medical professionals to approve the summary.
    • FR2: The program shall enable medical professionals to edit the summary.
    • FR3: The program shall allow medical professionals to submit the summary.
    • FR4: The program shall enable medical professionals to decline the summary.

Non-Functional Requirements

  • Patient Portal Requirements
    • NFR1: The program shall not interrupt the existing patient portal display.
    • NFR2: The program shall load the summary in at most 1 minute.
    • NFR3: The program shall display the summary in a format that is easy to read.
    • NFR4: The program shall display the summary using a simplified method free of medical jargon.
